Tutorial getDEVOData()

Funciton getDEVOData() retrieves data from DEVO and arranges the series in a MATLAB timetable. When multiple series are requested by the user, those are returned in a TxN timetable where N represents the number of series keys requested by the user (i.e., one column per requested series). In cases where one or multiple of the inputted series are not available, execution continues normally and the funciton returns all the series that are available, while producing a warning for the inexistent ones.
If a series is shorter than the rest of the requested series, the function returns the union (rather than the intersection) of the dates of the selected series, setting the absent observations into missing (NaN). Similarly, series of different frequencies can also be requested in the same function call, and missing observations (that are due to the frequency mismatch) are treated in the same manner. The resulting dataset can be automatically exported in an Excel spreadsheet using Name-Value pair 'ExportData'.

Function SYNTAX

%[y, Meta] = getDEVOData(serID, 'db',db,'StartDate',sdt,'EndDate',edt,'downsamle',freq,'dseries',false,'ExportData',exda);
 
%INPUTS:
%- 'serID': A Nx1 vector of strings containing the series-keys of the
% series to be downloaded.
%- 'db': A Nx1 vector of strings containing the database that
% corresponds to each of the series key in 'serID'.
%- 'ExportData': Logical value. When set to true, it exports the timetable with the series (contained in y) in an Excel spreadsheet
% named 'data' (i.e., 'data.xlsx'). The exported dataset is saved in the working directory. Default value is false.
% Alternatively, the function allows the user to provide a different path and filename for exporting the data,
% by passing as input a struct with fields: 'SavePath' and 'SaveName'.
%- 'StartDate': Specifies the series starting date. If the input is omitted (or set to []), the function returns the earliest
% starting date available in DEVO. The input should be specified as a datetime
% e.g. datetime(2021,3,31) for 31st March 2021.
%- 'EndDate': Specifies the series ending date. If the input is omitted (or set to []), the function returns the latest ending date
% available in DEVO. The input should be specified as a datetime.
%- 'dseries': Logical value. When set to true, output 'y' will return the requested series as a Dynare timeseries object i.e. dseries.
% If set to false, then 'y' is a timetable. Default value is false. NB: If there are series of different frequencies, then
% the dates in the resulting dseries object will be based on the highest frequency. For example, if Monthly and Quarterly
% series are requested, then the dseries dates will correspond to monthly dates (e.g., 2025M12).
%- 'connection': A connection object which can be used to establish connection to the desired database. To enable the function to connect
% to DEVO, the Cloudera ODBC driver must be installed on the user's workstation. By default, the function connects using
% the 'DEVO Impala 64-bit' DSN.
%- 'downsamle': An N-dimensional vector of vector of strings specifying the aggregation frequency for each series. By default, the function
% applies equally weighted averaging over all available periods. For example, converting a monthly series to quarterly will
% average all available months for each reference quarter. The allowed values are:
% - 'Q' for quarterly
% - 'M' for monthly
% NB: If a quarter contains only 2 of the 3 months then the average will be based on these months only. This default
% behavior can be changed (by setting NaNRaggedEdges=true) so that incomplete periods are set to NaN.
%
%OUTPUTS:
%- 'y': A timetable of size T-by-N containing the timeseries, where N is the number of the specified series in 'serID'.
% The dates in the resulted timetable are in DD/MM/YYYY format. If name-value pair 'dseries' is set to true, then
% y is returned as a dseries object.
%- 'Meta': A NxT-by-K sized table containing metadata information for the N requested variables. Note that different series have
% different matadata stored. Therefore, columns for which information is unavailable, will be empty.

By default the function exports the data in the working directory (pwd) in an Excel file named 'data.xlsx'.
[y, Meta] = getDEVOData(serid,'db',db,'ExportData',true);
The user can provide a custom path and filename for exporting the data, by passing as input to 'ExportData' a struct with two fields, 'SavePath' and 'SaveName'
exda = struct('SavePath','D:\', 'SaveName','data_v2');
[y, Meta] = getDEVOData(serid,'db',db, 'ExportData',exda);

(B) Request series from the same database

When all the requested data originate from the same database, then the name of the database can be passed as a single string (or char).
serid = ["PROJ.XXXF.W25.Q.N.U2.GAS", "PROJ.XXXF.W25.Q.N.U2.HIC", "PROJ.XXXF.W25.Q.N.U2.YER"];
db = ["crp_projections.proj"];
[y, Meta] = getDEVOData(serid,'db',db,'ExportData',false);
When all the requested data originate from the Projections database ('crp_projections'), then the user can even ommit input 'db'.
serid = ["PROJ.XXXF.W25.Q.N.U2.GAS", "PROJ.XXXF.W25.Q.N.U2.HIC","FIP.FIP0.G25.Q.C.I9.TIN"];
[y, Meta] = getDEVOData(serid, 'db',"", 'ExportData',false);
[y, Meta] = getDEVOData(serid, 'ExportData',false);

Chronologies of Euro Area Business Cycles by EABCN (Euro Area Business Cycle Network) Committee

%See: web('https://eabcn.org/dbc/peaksandtroughs/chronology-euro-area-business-cycles')
Recessions = ["1974 Q3", "1975 Q1"; "1980 Q1", "1982 Q3"; "1992 Q1", "1993 Q3"; "2008 Q1", "2009 Q2"; "2011 Q3", "2013 Q1"; "2019 Q4", "2020 Q2"];
Recessions = datetime(Recessions,'InputFormat','yyyy QQQ','Format','dd-MMM-yyyy');
NB: The EABCN Committee states that if a peak (trough) occurs during quarter P (T), the recession (expansion) starts at P+1 (T+1).
chrnlgies = Recessions + calquarters(1); %[Peak Trough] -> [Start, End] or Recession
EventDates = eventtable(chrnlgies(:,1),'EventEnds',chrnlgies(:,2), 'EventLabels',"Recession"); %create an events table
 
%Print the Start/End EABCN Euro area Recession dates
array2table(datetime(chrnlgies,'Format','yyyyQQQ'),'VariableNames',["Start","End"])

Custom Cloudera ODBC DNS

To setup the access to the different DEVO environments, follow the instruction below:
  • For DEVO in production envirnoment, see here: DEVO/DISC client package.
  • To access the acceptance DEVO environment, you need to create a custom DNS. See here: User Custom DNS configuration for FPM ACC DEVO.
By default, the function connects using the 'DEVO Impala 64-bit' DSN for the projections database and the 'DEVO-ACC' DSN for acceptance environment. If you have a different ODBC driver configuration in your workstation (for either the production or the acceptance environment), then you can use name-value pair 'connection' to pass a customized connection object to establish connection with the desired database.
For example, you can use the following syntax to create a connection object (cnxn) and then pass it manually in the function:
cnxn = database('DEVO-ACC','','');
y = getDEVOData("MST.BL.G25.Q.N.U2.YER", 'db',"lab_dlb_dge_fpm.mst", 'connection',cnxn);
For the IGAM roles and entitlements required to obtain access rights to the respective database please speak to your coordinator (- for FPM personel please see here).
